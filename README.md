# podcast-transcriber-jp

日本語ポッドキャストの「話者ごとに録音された音声トラック」をまとめて文字起こしする Windows 用ツールです。

ローカルで動きます。音声はインターネットには送信されません。

## はじめての方へ（インストール手順）

1. [Releases](../../releases) から **podcast-transcriber-jp.zip** をダウンロード
2. ダウンロードした ZIP を右クリック → **すべて展開** で解凍
3. 解凍後のフォルダの中にある **`start.bat`** をダブルクリック

これだけです。Python のインストールなど事前準備は不要です。

NVIDIA GeForce などの GPU が PC に載っている場合は自動的に GPU が使われ、10倍ほど高速になります。GPU がない場合は CPU で動作します（時間はかかりますが結果は同じです）。

### 「Windows によって PC が保護されました」と出たら

Microsoft が署名していない実行ファイルなので警告が出ます。

1. 青い画面の **「詳細情報」** をクリック
2. 下に出てきた **「実行」** をクリック

## 初回起動について

初回起動時のみ、文字起こしモデル（約 **3GB**）が自動でダウンロードされます。ネット環境にもよりますが **5〜15分** かかります。

ダウンロード先は ZIP 配下の `models/hf_cache/` です。2回目以降は再ダウンロード不要です。

## 画面構成

GUI を起動すると、上部に5つのタブがあります。

| タブ        | 役割                                                       |
|-------------|------------------------------------------------------------|
| **① 入力**      | 音声ファイルと用語リストを設定する。基本これだけ触れば動く |
| **② バッチ**    | 複数エピソードをまとめて夜間処理する                       |
| **③ 詳細設定**  | モデルや精度パラメータ（通常は変更不要）                   |
| **④ 手動retry** | 既存出力の特定区間だけ作り直す                             |
| **⑤ ログ**      | 処理の進捗ログ                                             |

画面の一番下に **「入力チェックして実行」** **「キューを順番に実行」** **「停止」** の3つのボタンが常に出ています。

## 基本の使い方（1エピソード処理）

1. `start.bat` をダブルクリック → GUI が開く
2. **① 入力** タブを開く
3. **「1. 話者別トラック」** の枠に、話者ごとの音声 (wav/mp3) を **ドラッグ&ドロップ** で2つ以上入れる
   - またはその下の **「音声を追加」** ボタンでファイル選択
4. 話者名（TRACK_1 / TRACK_2 等の初期値）を変えたい時:
   - リストで対象行をクリック
   - 下の **「選択した話者名」** 欄に名前を入力（例: `HORIMOTO`）
   - **「名前を反映」** ボタンを押す
5. **「2. 保存先」** で出力フォルダとファイル名を確認（自動入力済み、変更可）
6. **「3. 専門用語・固有名詞」** 欄に、固有名詞を **1行1語** で書く（任意・精度向上）
   - 例:
     ```
     堀元
     水野
     Wi-Fi
     ゆるコンピュータ科学ラジオ
     ```
7. 画面下の **「入力チェックして実行」** を押す
8. **⑤ ログ** タブで進捗を確認

処理時間の目安（72分の音声）:
- GPU 搭載PC: 30分〜1時間
- CPU のみ: 5〜10時間

## 用語リストを保存して再利用

- 「3. 専門用語」欄の右下の **「保存」** で .txt ファイルとして保存
- 別エピソードで **「読込」** から読み戻し可能
- ③ 詳細設定タブの **「hotwordsファイル」** に固定ファイルを指定しておくこともできる

## バッチ処理（複数エピソードを順次実行）

1. ① 入力タブで **1つ目** のエピソードを設定（話者トラック・保存先・用語）
2. ② バッチタブを開いて **「現在の設定を追加」** を押す
3. ① タブに戻って **2つ目** のエピソードを設定
4. ② タブで再度 **「現在の設定を追加」** （必要なだけ繰り返し）
5. 画面下の **「キューを順番に実行」** ボタンを押すと順番に処理される
6. キューは **「キュー保存」** で .json として保存し、次回 **「キュー読込」** で復元可能

長時間処理になるので、PCをスリープさせずに放置（夜間処理）するのに使います。

## 一部だけやり直す（手動retry）

文字起こし結果の特定区間が変だった時、その区間だけ再処理して上書きできます。

1. ④ 手動retry タブを開く
2. **「既存JSON」** で対象の `transcript_speaker_tracks.json` を選ぶ（または **「現在の出力をセット」**）
3. **「ASR対象音声/mix」** に `*_mix.wav` を指定（自動推定もされる）
4. **「範囲 秒」** に再処理したい区間を秒で入れる（例: start=120.0 end=180.0）
   - 空欄なら自動検出された「怪しい区間」だけが対象
5. **「手動retry実行」** を押す
6. 元JSONは保持され、`*.manual_retry.json` として別ファイルで出力

## 詳細設定（通常は変更不要）

③ 詳細設定タブの主要項目:

- **モデル**: 既定 `large-v3`（最高精度）。速度優先なら `medium` 以下
- **言語**: 既定 `ja`
- **基本プロンプト**: Whisperに渡す文脈ヒント（既定で日本語会話用テンプレートが入っている）
- **話者識別方式**: `v1` / **`v2`**（既定、推奨）
- **RMS/VAD**: `active dB` / `overlap dB` / `margin秒` — すべて `auto` のままで OK
- **クロストーク除去**: 既定ON。話者間のマイク漏れを抑える。OFF にする必要はほぼ無し

## 出力されるファイル

出力フォルダに以下が作成されます:

- `transcript_speaker_tracks.json` — 文字起こし本体（構造化データ）
- `transcript_speaker_tracks.clean.md` — 読みやすい本文
- `transcript_speaker_tracks.review.md` — 注釈つき詳細版
- `transcript_speaker_tracks.simple.txt` — シンプルなテキスト
- `transcript_speaker_tracks.log.txt` — 処理ログ

## 動作するPC

- **OS**: Windows 10 / 11（64bit）
- **メモリ**: 8GB 以上（16GB 推奨）
- **空き容量**: 5GB 以上
- **GPU**（任意）: NVIDIA GeForce などがあると自動で利用されます（CUDA 12.x 対応）

## トラブルシュート

**Q. 起動しない / 黒い画面が一瞬出て消える**
- ZIP を「すべて展開」ではなく、ZIP のままダブルクリックしていませんか? 必ず一度解凍してから `start.bat` を実行してください

**Q. 初回起動で「ダウンロード失敗」と出る**
- インターネット接続を確認してください。社内プロキシ環境では使えないことがあります
- 一度すべてのウィンドウを閉じて、再度 `start.bat` を実行してください

**Q. メモリ不足エラーになる**
- 他のアプリを閉じてください
- 1時間以上の長い音声は、短く分割してから処理することを検討してください
